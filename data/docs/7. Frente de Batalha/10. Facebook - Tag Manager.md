# Apostila: Google Tag Manager para Facebook e Tráfego Pago

---

## Resumo

Google Tag Manager (GTM) é uma ferramenta gratuita do Google que centraliza o gerenciamento de códigos de rastreamento (tags) em sites e aplicativos, eliminando a necessidade de alterar diretamente o código-fonte a cada modificação. O GTM facilita a instalação e o controle de pixels e scripts de diversas plataformas, como Facebook, Google Ads e Google Analytics, otimizando campanhas de tráfego pago. A estrutura do GTM é composta por containers, tags, triggers (gatilhos) e variáveis, que juntos permitem configurar quando e como os códigos são acionados. A publicação das alterações no GTM é essencial para que as configurações entrem em vigor no site.

---

## 1. Introdução ao Google Tag Manager (GTM)

### 1.1 O que é o Google Tag Manager?

- Ferramenta gratuita do Google para gerenciamento de tags em sites e aplicativos.
- Centraliza todos os códigos de rastreamento em um único local.
- Evita a necessidade de editar o código-fonte do site a cada alteração.
- Amplamente utilizado por profissionais de marketing digital e desenvolvedores.

### 1.2 Problema que o GTM resolve

- Cada plataforma de anúncios (Facebook, Google Ads, LinkedIn, Pinterest, etc.) exige um código específico (pixel).
- Sem o GTM, é necessário inserir manualmente cada código em cada site.
- Em múltiplos sites, a manutenção torna-se complexa e trabalhosa.
- O GTM instala um único código no site que gerencia todos os demais códigos configurados dentro dele.
- Alterações são feitas centralmente no GTM e refletidas em todos os sites com o código instalado.

## 2. Estrutura e Funcionamento do Google Tag Manager

### 2.1 Componentes principais do GTM

| Componente | Definição | Exemplo de Uso |
|------------|-----------|----------------|
| **Container** | Caixa que armazena todas as tags de um site ou app | Container para site, blog ou loja |
| **Tags** | Códigos que serão executados | Pixel do Facebook, Google Analytics, Google Ads |
| **Triggers (Gatilhos)** | Definem quando as tags serão ativadas | Ao carregar página, clicar botão, enviar formulário |
| **Variáveis** | Valores dinâmicos usados em tags ou triggers | URL da página, nome do botão clicado, ID do pixel |

- Para iniciantes, o foco principal é em containers, tags e triggers.
- Variáveis são usadas em configurações avançadas.

### 2.2 Lógica de funcionamento

- Criar um container para o site ou app.
- Inserir tags (códigos) dentro do container.
- Associar triggers que definem o momento de disparo da tag.
- Exemplo: Tag com código do Facebook acionada ao visitar uma URL específica.
- Quando o trigger é ativado, a tag envia dados para a plataforma correspondente.

## 3. Conceito de Pixel e sua Importância

- Pixel é um código inserido na página que rastreia ações do visitante.
- Funciona como um "dedo-duro", enviando dados para a plataforma de origem.
- Objetivo: rastrear visitantes e eventos para otimizar campanhas.
- Eventos comuns: PageView (visita à página), Lead (cadastro), Purchase (compra), ViewContent (visualização de conteúdo).

## 5. Como Criar e Configurar o Google Tag Manager

### 5.1 Criando a conta e o container

1. Acessar [Google Tag Manager](https://tagmanager.google.com) e fazer login com conta Google.
2. Criar nova conta:
   - Nome da conta (ex: nome do negócio ou projeto).
   - País de localização.
   - Endereço do site (URL).
   - Plataforma de destino: escolher "Web" para sites.
3. Aceitar termos de serviço e confirmar criação.
4. O sistema gera dois códigos para instalação:
   - Código para inserir na seção **head** do site.
   - Código para inserir na seção **body** do site.

### 5.2 Entendendo Head e Body

- **Head**: parte do código-fonte invisível ao usuário, localizada no topo da página.
- **Body**: corpo do código-fonte da página, também invisível ao usuário.
- Inserir o primeiro trecho do GTM no **head** e o segundo no **body**.

### 5.3 Instalando o código do GTM no site

- **WordPress / Elementor**: instalar plugin de Tag Manager, inserir os dois códigos nos campos indicados.
- **HotmartPages, Unbounce, LeadPages**: usar campos nativos para inserir códigos no head e body.
- **Sites personalizados**: solicitar ajuda de desenvolvedor para inserir códigos diretamente no código-fonte.

## 6. Instalando o Pixel do Facebook no Google Tag Manager

### 6.1 Métodos para instalação

#### Método 1: HTML Personalizado

1. No Facebook, acessar Gerenciador de Eventos.
2. Selecionar o pixel criado em Fontes de Dados.
3. Clicar em "Adicionar Eventos" → "Adicionar Nova Integração".
4. Escolher "Pixel da Meta" → "Configurar" → "Adicionar código manualmente".
5. Copiar o bloco de código gerado.

No GTM:

1. Acessar "Tags" → "Nova Tag".
2. Nomear a tag (ex: "Pixel do Facebook").
3. Selecionar "HTML Personalizado" e colar o código copiado.
4. Configurar acionamento (trigger).

#### Método 2: Template do Facebook Pixel

1. No GTM, acessar "Tags" → "Nova Tag".
2. Clicar em "Descubra mais tipos de tag".
3. Buscar e adicionar "Facebook Pixel".
4. Inserir o ID do pixel (encontrado no Gerenciador de Eventos do Facebook).
5. Selecionar o evento desejado (ex: PageView).
6. Configurar acionamento (trigger).

### 6.2 Configurando acionadores (triggers)

- Acionador define quando a tag será disparada.
- Sem acionador, a tag não será executada.

#### Acionador para todas as páginas

1. Na configuração da tag, clicar em acionamento.
2. Selecionar "All Pages" ou "Todas as Páginas".
3. Salvar.

#### Acionador para página específica

1. Criar novo acionador.
2. Nomear (ex: "Visita Página Vendas").
3. Selecionar tipo "Exibição de Página".
4. Marcar "Algumas exibições de página".
5. Configurar condição: `Page URL` contém `/PV0622` (exemplo).
6. Salvar.

## 7. Testando e Publicando as Configurações no GTM

### 7.1 Modo de visualização (Preview)

1. No GTM, clicar em "Visualizar" ou "Preview".
2. Inserir URL do site onde o GTM está instalado.
3. Clicar em "Conectar" ou "Connect".
4. Acompanhar tags disparadas na seção "Tags Fired".
5. Verificar se a tag do Facebook (ou outra) foi acionada corretamente.

### 7.2 Publicando alterações

- Salvar não é suficiente; é necessário publicar para aplicar as mudanças.
- Passos para publicar:
  1. Clicar em "Enviar" ou "Submit".
  2. Inserir nome e descrição das alterações.
  3. Confirmar publicação.
- Após publicação, as alterações passam a valer no site.

## 8. Benefícios do Google Tag Manager

- **Organização:** centraliza todos os códigos em um único local.
- **Facilidade:** permite alterações sem acesso direto ao código-fonte.
- **Segurança:** possibilita testes antes da publicação.
- **Independência:** não depende de programador para ajustes futuros.
- **Escalabilidade:** gerencia múltiplos sites e clientes com containers separados.

## 9. Aplicações Práticas e Exemplos

### 9.4 Outras integrações via GTM

- Ferramentas de mapa de calor (Hotjar, Clarity).
- Testes A/B (VWO).
- Ferramentas de chat (DivoChat, Tawk.to, Drift, Zendesk).

- Google Tag Manager é essencial para centralizar e facilitar o gerenciamento de pixels e scripts em sites.

- Containers organizam tags, triggers definem quando as tags disparam, e variáveis permitem configurações avançadas.
- Pixel do Facebook e Google Ads são os códigos mais comuns gerenciados via GTM.

- Publicar as alterações no GTM é obrigatório para que as mudanças tenham efeito.
- GTM elimina a necessidade de editar código-fonte diretamente, facilitando ajustes e manutenção.
- Pode ser usado em diversas plataformas, desde WordPress até HotmartPages e sites personalizados.
