## Aula 10: Facebook - Google Tag Manager (GTM)

## Resumo

Nesta aula, aprendemos o que é o Google Tag Manager (GTM), sua importância para o gerenciamento centralizado de códigos de rastreamento (tags) em sites e aplicativos, e como configurá-lo para integrar o pixel do Facebook e outras ferramentas essenciais para campanhas de tráfego pago. O GTM simplifica a implementação e manutenção de múltiplos pixels e scripts, evitando a necessidade de alterar diretamente o código-fonte do site a cada modificação. Também exploramos a estrutura do GTM, seus principais componentes (containers, tags, triggers e variáveis), e o passo a passo para criar uma conta, configurar um container, instalar o código no site e criar tags acionadas por eventos específicos.

---

## 1. Introdução ao Google Tag Manager (GTM)

### 1.1 O que é o Google Tag Manager?

- Ferramenta gratuita do Google para gerenciamento de tags (trechos de código) em sites e aplicativos.
- Permite centralizar todos os códigos de rastreamento em um único local, sem necessidade de alterar o código-fonte do site a cada mudança.
- Amplamente utilizado por profissionais de marketing digital, desenvolvedores e gestores de tráfego em todo o mundo.
- Facilita a instalação, alteração e remoção de pixels e scripts de diversas plataformas (Facebook, Google Ads, LinkedIn, Pinterest, Taboola, Outbrain, entre outros).

### 1.2 Problema que o GTM resolve

- Sem o GTM, cada plataforma exige a inserção manual de seu código no site.
- Em múltiplos sites, essa tarefa se torna repetitiva e suscetível a erros.
- Toda alteração exige acesso individual a cada site para modificar os códigos.
- O GTM centraliza esses códigos em um container, que é gerenciado por um único código instalado no site.
- Alterações feitas no GTM são refletidas automaticamente em todos os sites que possuem o código do GTM instalado.

## 2. Estrutura e Funcionamento do Google Tag Manager

### 2.1 Componentes principais do GTM

| Componente | Definição | Exemplo de Uso |
|------------|-----------|----------------|
| **Container** | Caixa onde são adicionadas todas as tags. Cada site ou app possui seu próprio container. | Container para site, blog ou loja. |
| **Tags** | Códigos que você deseja executar, como pixels ou scripts. | Pixel do Facebook, Google Analytics, Google Ads. |
| **Triggers (Gatilhos)** | Definem quando cada tag será ativada. | Ao carregar página, clicar em botão, enviar formulário. |
| **Variáveis** | Valores dinâmicos usados em tags ou triggers. | URL da página, nome do botão clicado, ID do pixel. |

- Para iniciantes, os componentes mais usados são **containers**, **tags** e **triggers**.
- Variáveis são usadas em configurações mais avançadas.

### 2.2 Lógica de funcionamento

- Dentro do container, cria-se uma tag que contém o código de rastreamento.
- Cada tag é associada a um trigger que define o evento que a ativa.
- Exemplo: tag do pixel do Facebook acionada ao visitar uma URL específica.
- Quando o trigger é ativado, a tag envia os dados para a plataforma correspondente.

## 3. Conceito de Pixel e Eventos

### 3.1 O que é um Pixel?

- Código inserido em uma página que funciona como um "dedo-duro" ou espião.
- Captura e envia dados sobre as ações do visitante para a plataforma de origem.
- Permite rastrear visitantes e eventos para otimizar campanhas publicitárias.

### 3.2 Eventos comuns do Pixel do Facebook

| Evento       | Descrição                                  |
|--------------|--------------------------------------------|
| **PageView** | Visita à página                            |
| **ViewContent** | Visualização de conteúdo específico (ex: página de vendas, vídeo) |
| **Lead**     | Cadastro em formulário                     |
| **Purchase** | Compra realizada                           |

- Eventos personalizados também podem ser criados para rastrear ações específicas, como clique em botão, preenchimento de formulário, clique em WhatsApp, visualização parcial de vídeo, etc.
- Facebook oferece ferramenta visual para criação de eventos personalizados.

## 4. Principais Pixels e Tags Usadas no GTM

| Plataforma       | Objetivo / Uso                                                                                  |
|------------------|------------------------------------------------------------------------------------------------|
| **Facebook Pixel** | Rastrear visitantes e eventos para otimizar campanhas no Facebook e Instagram.                  |
| **Google Ads**    | Medir conversões e otimizar campanhas, por exemplo, rastrear percentual de vídeo assistido.    |
| **Google Analytics** | Analisar comportamento do usuário: páginas acessadas, cliques, scroll, taxa de rejeição, jornada até compra. |
| **Eventos Personalizados** | Eventos específicos criados conforme necessidade, além dos padrões.                        |
| **Hotjar**        | Mapa de calor e gravação de sessões para análise de comportamento do usuário.                   |
| **Clarity (Microsoft)** | Mapa de calor e insights sobre comportamento do usuário.                                    |
| **VWO**           | Testes A/B para comparar versões de páginas e otimizar conversão.                               |
| **Ferramentas de Chat** | DivoChat, Tawk.to, Drift, Zendesk (Zupin) para atendimento em tempo real.                   |

## 5. Como Criar e Configurar o Google Tag Manager

### 5.1 Criando a Conta e o Container

1. Acesse [Google Tag Manager](https://tagmanager.google.com) e faça login com sua conta Google.
2. Clique em **Criar Conta**.
3. Preencha:
   - Nome da conta (ex: "Aula de Tráfego").
   - País (ex: Brasil).
   - Nome do container (ex: domínio do site, como "vendatodosantodia.com.br").
   - Plataforma de destino: escolha **Web** para sites.
4. Aceite os termos e confirme a criação.
5. O GTM gera dois trechos de código:
   - Um para ser inserido na seção **head** do site.
   - Outro para ser inserido na seção **body** do site.

### 5.2 Entendendo Head e Body

- **Front-end**: o que o visitante vê na tela.
- **Back-end**: código-fonte do site, invisível ao usuário.
- **Head**: seção superior do código-fonte, onde deve ser colocado o primeiro trecho do GTM.
- **Body**: corpo do código-fonte, onde deve ser colocado o segundo trecho do GTM.

### 5.3 Instalando o código do GTM no site

- **WordPress / Elementor**: instalar plugin de GTM, inserir os dois códigos nos campos indicados.
- **HotmartPages, Unbounce, LeadPages**: geralmente possuem campos nativos para inserir códigos no head e body.
- **Sites personalizados**: requer auxílio de desenvolvedor para inserir os códigos diretamente no código-fonte.

## 6. Como Configurar o Pixel do Facebook no GTM

### 6.1 Método 1: HTML Personalizado

1. No Facebook, acesse o **Gerenciador de Eventos**.
2. Vá em **Fontes de Dados** e selecione seu pixel.
3. Clique em **Adicionar Eventos** → **Adicionar Nova Integração**.
4. Escolha **Pixel da Meta** → **Configurar** → **Adicionar código manualmente**.
5. Copie o bloco de código gerado.
6. No GTM, vá em **Tags** → **Nova Tag**.
7. Nomeie a tag (ex: "Pixel do Facebook").
8. Selecione **HTML Personalizado** e cole o código.
9. Configure o acionamento (trigger).

### 6.2 Método 2: Template do Facebook Pixel

1. No GTM, vá em **Tags** → **Nova Tag**.
2. Clique em **Descubra mais tipos de tag**.
3. Busque por **Facebook Pixel** e adicione ao espaço de trabalho.
4. Insira o ID do pixel (encontrado no Gerenciador de Eventos do Facebook).
5. Selecione o evento desejado (ex: PageView).

## 7. Configurando Acionadores (Triggers)

- O acionador define quando a tag será disparada.
- Sem acionador, a tag não será executada.

### 7.1 Acionador para todas as páginas

- Na configuração da tag, clique em acionamento.
- Selecione **All Pages** (Todas as Páginas).
- Salve.

### 7.2 Acionador para página específica

1. Crie um novo acionador.
2. Dê nome descritivo (ex: "Visita Página Vendas").
3. Tipo: **Exibição de Página**.
4. Marque **Algumas exibições de página**.
5. Configure a condição:  
   - **Page URL**  
   - **contém**  
   - "/PV0622" (exemplo de trecho da URL da página de vendas).
6. Salve o acionador.

---

## 8. Testando e Publicando as Configurações

### 8.1 Testando com o modo de visualização

1. No GTM, clique em **Visualizar** ou **Preview**.
2. Insira a URL do site onde o GTM está instalado.
3. Clique em **Conectar**.
4. A página será aberta em modo de depuração.
5. Verifique na seção **Tags Fired** se as tags foram disparadas corretamente.

### 8.2 Publicando as alterações

- Salvar no GTM não aplica as alterações no site.
- Para aplicar, clique em **Enviar** ou **Submit**.
- Insira um nome e descrição para a versão.
- Confirme a publicação.

## 9. Benefícios do Google Tag Manager

- **Organização:** todos os códigos centralizados em um único local.
- **Facilidade:** alterações feitas sem acesso ao código-fonte.
- **Segurança:** possibilidade de testar antes de publicar.
- **Independência:** não depende de programador para alterações simples.
- **Escalabilidade:** gerencia múltiplos pixels e scripts em vários sites.

## 10. Exemplos Práticos e Casos de Uso

- Instalar pixel do Facebook para rastrear eventos como PageView, Lead, Purchase.
- Configurar Google Ads para medir conversões e otimizar campanhas.
- Usar Google Analytics para análise detalhada do comportamento do usuário.
- Criar eventos personalizados para acompanhar cliques em botões, preenchimento de formulários, interações específicas.
- Integrar ferramentas de mapa de calor como Hotjar e Clarity para insights de navegação.

- Gerenciar códigos de ferramentas de chat (DivoChat, Tawk.to, Drift, Zendesk).

## 11. Pontos-Chave

- O Google Tag Manager é uma ferramenta essencial para gerenciar múltiplos códigos de rastreamento de forma centralizada e eficiente.
- A instalação do GTM no site é feita uma única vez, e todas as alterações futuras são feitas dentro da plataforma.
- Containers organizam as tags, triggers definem quando as tags são disparadas, e variáveis fornecem dados dinâmicos para configurações avançadas.
- O pixel do Facebook é fundamental para rastrear visitantes e eventos, otimizando campanhas no Facebook e Instagram.
- É possível configurar tags para disparar em todas as páginas ou em páginas específicas, conforme a estratégia.
- Testar as tags antes de publicar evita erros e garante o funcionamento correto.
- O GTM elimina a dependência de programadores para alterações simples em códigos de rastreamento.
- Além do Facebook, o GTM suporta diversas outras ferramentas e pixels, ampliando o controle sobre as estratégias digitais.
- A configuração do GTM é intuitiva e acessível mesmo para iniciantes, com muitos recursos para facilitar a implementação.
