# Módulo 7: Frente de Batalha  
## Aula 26: Facebook - Integração com a Hotmart

---

## Resumo

Nesta aula, foi apresentado o passo a passo detalhado para integrar o pixel do Facebook (Meta) com a plataforma Hotmart, permitindo o rastreamento preciso de eventos de conversão, como initiate checkouts e vendas realizadas. A integração possibilita mensurar resultados, otimizar campanhas e criar públicos personalizados com base em dados reais de comportamento de compra. Além do pixel, a configuração da API de Conversões foi abordada para garantir maior precisão e redundância no envio dos dados. Também foram discutidas as melhores práticas para configuração dos eventos, métodos de pagamento e valores das transações.

---

## Conteúdo Detalhado

### 1. Introdução à Integração Facebook Pixel + Hotmart

A integração do pixel do Facebook com a Hotmart é fundamental para profissionais de tráfego pago que vendem produtos digitais. Ela conecta a plataforma de anúncios às conversões registradas na Hotmart, permitindo:

- Acompanhamento dos eventos desde o acesso à página de pagamento (initiate checkout) até a confirmação da venda.
- Mensuração precisa dos resultados das campanhas.
- Otimização do algoritmo do Facebook com dados reais.
- Criação de públicos personalizados e públicos semelhantes (Lookalike) baseados em compradores reais.

---

### 2. Pré-requisitos para a Integração

Antes de iniciar a configuração, certifique-se de possuir:

- Conta ativa na Hotmart.
- Produto criado e configurado na Hotmart.
- Acesso ao Gerenciador de Eventos do Facebook (Meta Business Suite).
- Pixel do Facebook previamente criado.

---

### 3. Acessando a Ferramenta de Pixel de Rastreamento na Hotmart

**Passo a passo:**

1. Faça login na sua conta Hotmart.
2. No menu lateral esquerdo, clique em **Ferramentas**.
3. Clique em **Ver todas** para visualizar todas as ferramentas disponíveis.
4. Utilize o campo de busca para localizar a funcionalidade **Pixel de Rastreamento**.
5. Ao acessar, selecione o produto que deseja configurar (cada produto tem configurações individualizadas).
6. Após selecionar o produto, a Hotmart exibirá as opções de integração disponíveis.

---

### 4. Plataformas Compatíveis para Integração na Hotmart

A Hotmart oferece integração com diversas plataformas para rastreamento e análise, entre elas:

| Plataforma           | Finalidade                                  |
|---------------------|---------------------------------------------|
| Facebook + Instagram | Rastreamento de conversões para Meta Ads    |
| Google Ads          | Rastreamento de conversões para Google Ads  |
| Google Analytics 4  | Análise de comportamento e métricas          |
| TikTok Ads          | Rastreamento de conversões para TikTok       |
| Bing Ads            | Rastreamento para Microsoft Advertising      |
| Taboola             | Rastreamento para campanhas de native ads    |
| Outbrain            | Rastreamento para campanhas de native ads    |
| Twitter (X) Ads     | Rastreamento de conversões para Twitter (X)  |

**Nesta aula, o foco é a integração com Facebook + Instagram.**

---

### 5. Obtendo o ID do Pixel no Facebook

O ID do pixel é o código identificador único que conecta sua conta de anúncios às conversões da Hotmart.

**Como localizar o ID do pixel:**

1. Acesse o **Gerenciador de Eventos** no Facebook (Meta Business Suite).
2. Na seção **Fontes de Dados**, selecione o pixel desejado.
3. O ID do pixel aparecerá na lateral direita da tela (sequência numérica).
4. Copie esse ID.
5. Volte para a Hotmart e cole o ID no campo correspondente.
6. Clique em **Continuar** para avançar.

---

### 6. Configuração dos Eventos de Rastreamento na Hotmart

Você deve selecionar quais eventos a Hotmart enviará para o pixel do Facebook.

| Evento                         | Descrição                                           | Recomendação       |
|-------------------------------|----------------------------------------------------|--------------------|
| Vendas realizadas              | Dispara quando uma compra é concluída               | Obrigatório        |
| Visitas na página de pagamento (Initiate Checkout) | Dispara quando alguém acessa o checkout            | Obrigatório        |
| Visitas na página do produto Hotmart | Dispara em acessos à página nativa do produto na Hotmart | Opcional*          |

> *Nota:* A página do produto Hotmart refere-se à página nativa da Hotmart, não à sua página de vendas personalizada. Caso utilize página externa, pode deixar desmarcado.

---

### 7. Configuração do Método de Pagamento para Confirmação de Venda

Você deve escolher quais métodos de pagamento serão contabilizados como conversão:

| Opção                  | Métodos Incluídos                      | Comportamento                                               |
|------------------------|--------------------------------------|-------------------------------------------------------------|
| Todos os métodos       | Cartões, boletos, Pix, saldo Hotmart | Contabiliza todas as transações, incluindo boletos e Pix não pagos |
| Apenas pagamentos imediatos | Cartões de crédito, débito e saldo Hotmart | Contabiliza somente transações com confirmação instantânea  |

**Recomendação:** Marcar apenas **pagamentos imediatos** para evitar que boletos ou Pix pendentes sejam contabilizados como vendas, o que prejudicaria a precisão da mensuração e atribuição.

---

### 8. Diferenciação de Eventos por Tipo de Pagamento

A Hotmart permite diferenciar eventos conforme o método de pagamento:

- Ao ativar essa opção, a Hotmart envia um evento específico chamado **Payment Generated** para pagamentos não imediatos (boletos, Pix).
- Isso permite:
  - Identificar boletos gerados versus pagos.
  - Criar estratégias de remarketing específicas para quem gerou boleto mas não pagou.
  - Manter análise de dados segmentada e organizada.

---

### 9. Configuração do Valor da Transação

Você pode escolher qual valor será enviado junto ao evento de compra:

| Opção                  | Funcionamento                                           |
|------------------------|--------------------------------------------------------|
| Valor da transação     | Envia o valor real pago pelo cliente                    |
| Valor específico definido | Envia um valor fixo predeterminado                      |

**Recomendação:** Utilizar o **valor da transação** para garantir que o Facebook receba dados precisos, especialmente quando há ofertas, descontos ou diferentes preços para o mesmo produto.

---

### 10. Configuração da API de Conversões

A API de Conversões é uma camada adicional que envia dados diretamente do servidor da Hotmart para o Facebook, complementando o rastreamento via navegador.

**Benefícios:**

- **Redundância:** Eventos enviados via navegador e servidor.
- **Maior precisão:** Reduz perdas causadas por bloqueadores de anúncios e configurações de privacidade.
- **Parâmetros adicionais:** Envio de informações detalhadas como método de pagamento, valor, e dados para identificar o comprador.

---

### 11. Obtenção e Inserção do Token de Acesso da API no Hotmart

**Passo a passo para gerar o token:**

1. No Facebook, acesse o **Gerenciador de Eventos**.
2. Vá para **Fontes de Dados** e selecione o pixel.
3. Clique em **Configurações** do pixel.
4. Role a página até encontrar a seção de geração de token.
5. Clique em **Gerar Token** para criar um código único.
6. Copie o token gerado.
7. Volte para a Hotmart e cole o token no campo correspondente.
8. Utilize a opção de **Verificação do Token** para confirmar a conexão (indicada por sinal verde).
9. Clique em **Salvar** para finalizar.

**Importante:**  
- Guarde uma cópia do token em local seguro, pois ele pode mudar com o tempo.  
- Ter um backup facilita futuras verificações e atualizações.

---

### 12. Considerações Finais da Integração

- Após a configuração, a Hotmart enviará automaticamente para o Facebook todos os eventos de **Initiate Checkout** e **vendas realizadas**, com parâmetros adicionais configurados.
- Essa integração é essencial para:
  - Mensurar com precisão o ROI das campanhas.
  - Alimentar o algoritmo do Facebook para otimização.
  - Construir públicos personalizados e semelhantes baseados em dados reais.
- Apesar da simplicidade do processo, sua importância estratégica é alta para decisões fundamentadas em dados confiáveis.

## Pontos-Chave

- A integração do pixel Facebook + Hotmart é essencial para rastreamento e otimização de campanhas de tráfego pago.
- Sempre selecione apenas pagamentos imediatos para evitar distorção na mensuração.
- Diferenciar eventos por método de pagamento permite estratégias de remarketing mais eficientes.
- Utilizar o valor real da transação garante dados precisos mesmo com promoções.
- A API de Conversões complementa o rastreamento via navegador, aumentando a confiabilidade dos dados.
- O token da API deve ser guardado com segurança e verificado regularmente.
- Uma configuração correta impacta diretamente na qualidade da análise e no desempenho das campanhas.

## Exemplos e Analogias Utilizadas

- O instrutor comparou pagamentos imediatos (cartão, débito) com pagamentos não imediatos (boletos, Pix) para explicar a importância de selecionar corretamente os métodos de pagamento para evitar "sujar" a atribuição das vendas.
- A API de Conversões foi explicada como um envio de dados "via servidor", garantindo que mesmo bloqueadores de anúncios não impeçam o registro dos eventos.
